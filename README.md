# Topaz-Next

> **非商用开源项目** | 基于 LightGBM + 技术特征因子的 A 股量化分析与自动决策系统。
>
> 仅供学习研究使用，**禁止用于任何商业目的或实际资金交易**。作者不对任何投资损失承担责任。

---

## 项目简介

Topaz-Next 是一个面向个人研究者的 A 股量化分析系统。以**沪深300成分股**为选股池，通过特征工程构建 45+ 技术因子，输入 LightGBM 模型预测每只股票的短期上涨概率，在成分股内部优选标的、动态调仓，试图获取相对沪深300指数的**超额收益（Alpha）**。

## 核心流程

```
09:45  扫描沪深300成分股 → 获取行情 → 特征工程 → LightGBM 预测概率
       └─ 市场环境判断 → Slack 直发：Top 5 买入 + Bottom 7 回避

10:00  生成投资决策（买入/卖出/持有）→ 更新虚拟组合

14:30  执行调仓 → 发送完整交易报告到 Slack
```

## 模型与特征

### 特征工程（45+ 因子）

`feature_engineer.py` 从 OHLCV 数据自动生成以下特征：

| 类别 | 代表因子 | 数量 |
|------|---------|------|
| 价格位置 | 相对均线位置、相对高低点位置 | ~8 |
| 动量 | 多周期收益率、时间序列动量、动量加速度、均线交叉 | ~12 |
| 波动率 | 历史波动率、EWMA 波动率、ATR、偏度、峰度 | ~10 |
| 成交量 | 量比、OBV、Amihud 非流动性 | ~5 |
| 技术指标 | RSI、MACD、KDJ、布林带、CCI、ADX、Williams %R | ~12 |
| 价格形态 | K线实体、锤子线、射击之星、吞没形态 | ~5 |
| 统计因子 | 均值回归 Z-Score、最大回撤、夏普代理、尾部风险 | ~6 |

此外包含**大盘指数因子**（沪深300收益率、均线位置、波动率）和**相对强度因子**（个股超额收益、Beta）。

### 模型

生产路径使用 **LightGBM 分类模型**。模型输出每只股票未来上涨的**概率**（0~1），用于后续决策。

模型训练脚本位于 `retrain_model.py` 和 `retrain_model_walkforward.py`（需手动运行）。

## 策略逻辑与参数

### 1. 市场环境判断

根据沪深300的 **20日收益** 和 **市场上涨比例** 判断环境：

| 环境 | 条件 | 模型置信度 | 建议仓位 |
|------|------|-----------|---------|
| 反弹 | 上涨>55% 且 20日收益<-2% | 90% | 95% |
| 回调 | 上涨<45% 且 20日收益>2% | 80% | 80% |
| 牛市 | 上涨>55% 且 20日收益>0 | 70% | 70% |
| 熊市 | 上涨<45% 且 20日收益<-2% | 60% | 20% |
| 震荡 | 其他 | 50% | 50% |

### 2. 买入策略

**综合评分**：
```
score = probability × (1 + predicted_return / 10)
```

动态买入门槛和仓位（按环境）：

| 环境 | 买入门槛 | 单股上限 | 高分仓位 |
|------|---------|---------|---------|
| 牛市 | 概率>55% | 25% | 25% |
| 反弹 | 概率>55% | 20% | 20% |
| 震荡 | 概率>55% | 10% | 10% |
| 熊市 | 概率>85% | 10% | 10% |

**抄底加仓**（仅亏损持仓）：
- 震荡市：亏损>8% + 评分>0.75 → 加仓 6%
- 其他：亏损>5% + 评分>0.7 → 加仓 15%

### 3. 卖出策略

| 触发条件 | 说明 | 阈值（按环境） |
|---------|------|--------------|
| 止损 | 亏损超过阈值 | 熊市-4% / 震荡-6% / 其他-8% |
| 止盈 | 盈利且评分下降 | 熊市/震荡>8%+概率<0.7 / 其他>15%+概率<0.7 |
| ML回避 | 模型建议回避 | 熊市<0.5 / 震荡<0.4 / 其他<0.35 |
| 换仓 | 盈利+有更好标的 | 盈利>5%且分差>0.15（牛市>0.2） |
| 集中度风控 | 单股>25%且低评分亏损 | 触发减仓 |

### 4. 仓位管理

- **禁止孖展**：买入前严格校验现金余额
- **动态总仓位**：根据市场环境限制（20%~95%）
- **单股上限**：防止过度集中（10%~25%）
- **现金预留**：熊市最低 20%，震荡市 50%

### 运行流程

```
09:45  daily_report.sh
       └─ python ml_stock_analysis_ensemble.py

10:00  daily_decision.sh
       └─ python daily_decision.py
       └─ 代码直发 Slack：Top 5 评分最高 + Bottom 7 评分最低
       └─ 生成买入/卖出/持有决策 → 更新 virtual_portfolio.json

14:30  trade_execute.sh
       └─ python daily_decision.py --execute
       └─ 执行调仓 → 发送完整交易报告到 Slack
```

### 交易日检查机制

系统内置多层校验，非交易日自动跳过：
- **Shell 层**：脚本检查周末（`date +%w`）
- **Python 层**：`trading_utils.py` 的 `is_trading_day()` 检查节假日
- **执行层**：非交易时段阻止交易并记录日志

节假日数据位于 `trading_utils.py` 中的 `holidays_2026` 列表，需每年更新。

## 投资风险警示

⚠️ **本系统仅供学习研究，不构成任何投资建议。**

1. **模型局限**：基于历史数据训练，市场结构变化可能导致模型失效。A 股免费数据源存在 1-5 分钟延迟。
2. **虚拟组合 ≠ 实盘**：未考虑滑点、冲击成本、交易费用、印花税、融资融券限制等实盘因素。
3. **回撤风险**：历史上出现过 **-4.7% 的最大回撤**，熊市环境下模型置信度最低，可能连续亏损。
4. **集中度风险**：单只股票仓位最高可达 25%，存在特定行业集中波动风险。
5. **技术风险**：API 可能因网络问题或服务商调整而失效；节假日配置错误可能导致非预期交易。
6. **合规声明**：**禁止用于任何商业目的**，包括但不限于代客理财、收费咨询、基金产品等。使用本项目进行实盘交易需自行承担全部法律责任和经济损失。
